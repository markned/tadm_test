# Промпт для реализации режима экзамена

## Задача
Реализовать режим экзамена с отключенными подсказками и немедленной обратной связью, заменяющий стандартный режим тестирования.

## Требования к режиму экзамена

### 1. Ограничения функциональности в режиме экзамена
- **Study Mode полностью отключен**: 
  - Вкладка/кнопка переключения в Study Mode должна быть скрыта или неактивна
  - При включенном режиме экзамена переход в Study Mode невозможен
  
- **Подсказки отключены**:
  - Убрать функциональность hints (подсказки с подсветкой правильных ответов)
  - Удалить любые индикаторы, показывающие правильность ответа до завершения теста
  - Убрать кнопки или функциональность, связанную с подсказками

- **Немедленная обратная связь отключена**:
  - После выбора ответа не показывать, правильный ли ответ (зеленый/красный индикатор)
  - Статус ответа (correct/wrong) не должен отображаться до завершения теста
  - Визуальные индикаторы правильности ответа должны быть скрыты во время прохождения

- **Список вопросов без индикации статуса**:
  - В боковой панели со списком вопросов убрать все индикаторы правильности (StatusDot)
  - Не показывать цветовую маркировку вопросов (зеленый/красный/серый)
  - Список должен показывать только номера вопросов и текст вопроса
  - Можно оставить индикатор текущего активного вопроса (например, синяя рамка)

### 2. Результаты после завершения экзамена

#### Статистика по категориям:
- После завершения экзамена показывать детальную статистику по категориям:
  - Для каждой категории отображать: название категории, количество правильных ответов, общее количество вопросов в категории, процент правильных ответов
  - Формат отображения: таблица или карточки с категориями
  - Пример: "System Administration: 8/10 (80%)"
  - Использовать функцию `formatCategoryName()` для корректного отображения названий категорий

#### Просмотр неправильных ответов:
- После завершения экзамена добавить кнопку/секцию "Review Wrong Answers" или "View Incorrect Answers"
- При нажатии показывать список всех вопросов с неправильными ответами:
  - Отображать текст вопроса
  - Показывать выбранный пользователем ответ (помечен как неправильный)
  - Показывать правильный ответ (с выделением/подсветкой)
  - Показывать explanation, если оно есть
  - Форматирование должно быть понятным и четким

### 3. Настройки режимов работы

#### Режим с повторяющимися вопросами (Repeat Wrong Until Correct):
- Оставить существующий функциональность режима повторения неправильных ответов
- Сделать этот режим **дефолтным** (по умолчанию включен)
- При старте нового теста этот режим должен быть включен

#### Переключение между режимами:
- Изменить текущий свитч (toggle) с режима "Repeat wrong until correct" на свитч между:
  - **Режим повторения** (Repeat Mode) - дефолтный
  - **Режим экзамена** (Exam Mode)
- UI для переключения:
  - Radio buttons или toggle switch
  - Четкие метки: "Repeat Mode" и "Exam Mode"
  - Подсказка/описание каждого режима при наведении или рядом

### 4. Технические детали реализации

#### Состояние (State):
- Добавить новое состояние `examMode: boolean` в состояние компонента
- Это состояние должно влиять на:
  - Видимость Study Mode
  - Отображение подсказок
  - Отображение статуса ответов в реальном времени
  - Отображение результатов

#### Логика проверки ответов:
- В режиме экзамена ответы все равно должны проверяться и сохраняться
- Но визуальная обратная связь должна быть отключена
- Статус (correct/wrong) должен быть доступен только после завершения

#### Статистика по категориям:
- Создать функцию для группировки вопросов по категориям
- Для каждой категории подсчитать:
  - Общее количество вопросов
  - Количество правильных ответов
  - Процент правильных ответов
- Отобразить в удобном формате (таблица, карточки, прогресс-бары)

#### Результаты экзамена:
- Создать новый компонент/секцию для отображения результатов экзамена
- Включить:
  - Общую статистику (общий счет, процент)
  - Статистику по категориям
  - Кнопку для просмотра неправильных ответов

### 5. UI/UX соображения

#### Режим экзамена:
- Четко обозначить, что включен режим экзамена (бейдж, иконка, текст)
- При переходе в режим экзамена показать предупреждение о том, что подсказки отключены
- Возможно, добавить подтверждение при переключении в режим экзамена

#### Результаты:
- Сделать результаты читабельными и хорошо структурированными
- Использовать цвета для категорий (можно взять из существующей цветовой схемы)
- Добавить возможность экспорта результатов (опционально, для будущего)

### 6. Тестирование

Убедиться, что:
- В режиме экзамена Study Mode недоступен
- Подсказки не работают
- Статус ответов не показывается во время теста
- После завершения показывается статистика по категориям
- Можно просмотреть неправильные ответы
- Режим повторения работает как раньше и является дефолтным
- Переключение между режимами работает корректно

## Приоритеты реализации

1. **Высокий приоритет:**
   - Добавление состояния examMode
   - Отключение Study Mode в режиме экзамена
   - Отключение подсказок и немедленной обратной связи
   - Скрытие статуса в списке вопросов

2. **Средний приоритет:**
   - Статистика по категориям после завершения
   - Просмотр неправильных ответов

3. **Низкий приоритет:**
   - UI улучшения и предупреждения
   - Оптимизация отображения результатов

## Примечания

- Сохранить всю существующую функциональность для режима повторения
- Убедиться, что существующие тесты продолжают работать
- Код должен быть читаемым и поддерживаемым
- Добавить комментарии к новому функционалу
